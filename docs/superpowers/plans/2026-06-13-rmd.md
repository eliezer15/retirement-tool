# RMD (Required Minimum Distributions) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Model IRS Required Minimum Distributions from the Traditional account starting at age 73 — forcing a top-up withdrawal when planned spending falls short of the RMD, adding the extra cash to net spendable, and surfacing two new columns in the table.

**Architecture:** All changes are in `app.js` and `app.css`. The engine gains an RMD constants table, a `tradBalStartOfYear` tracker, and a lump year-end top-up step. The year result object gains `rmdRequired` and `rmdActual` fields. The render layer adds two table columns styled in purple.

**Tech Stack:** Vanilla JS, single `app.js`.

```js
// Source: IRS Publication 590-B, Table III (Uniform Lifetime).
// Ages below 73 are not subject to RMDs. Ages above 120 use the last known factor.

   73: 26.5,  74: 25.5,  75: 24.6,  76: 23.7,  77: 22.9,

  113: 2.9,  114: 2.6,  115: 2.4,  116: 2.1,  117: 1.9,

};

/** Return the RMD distribution period for a given age.
 * Returns null for ages below 73 (no RMD required).
 * Uses last known factor for ages above the table max. */
function getRmdFactor(age) {
  if (age < 73) return null;
  const factor = RMD_UNIFORM_TABLE[age];
  if (factor !== undefined) return factor;
  // Age beyond table — use the last entry
  return RMD_UNIFORM_TABLE[120];
}
```

Replace with:
```js
    const newYear = Math.floor((m + 1) / 12);
    if (newYear > currentYear) {
      // age = age at START of this year (the age you are during these 12 months).
      // Months m=(newYear-1)*12 .. newYear*12-1 cover ageFloat from startAge+(newYear-1) to startAge+newYear.
      const age = cfg.startAge + newYear - 1;

      // --- RMD top-up (Option A: lump year-end forced withdrawal) ---
      // Apply BEFORE computing taxes so the extra ordinary income is taxed correctly.
      const rmdFactor = getRmdFactor(age);
      const rmdRequired = rmdFactor != null ? tradBalStartOfYear / rmdFactor : 0;
      const rmdShortfall = Math.max(0, rmdRequired - yearTradWithdrawn);
      const rmdActual = Math.min(rmdShortfall, bal.traditional); // capped at available balance
      if (rmdActual > 0) {
        bal.traditional   -= rmdActual;
        yearOrdinaryIncome += rmdActual; // RMD is fully ordinary income
        yearWithdrawn      += rmdActual; // adds to spendable cash
        // RMD top-ups are post-59.5 by definition (RMD starts at 73) — no penalty
      }

      const { ordinaryTax, capGainsTax } = computeAnnualTax(yearOrdinaryIncome, yearCapGainsRealized);
      const penaltyTax = yearPenaltyBase * 0.10;
      const totalBalance = bal.traditional + bal.roth + bal.taxable;
      years.push({

        penaltyTax,

        rmdActual,    // extra forced withdrawal on top of planned withdrawals
        totalTax: ordinaryTax + capGainsTax + penaltyTax,

      yearCapGainsRealized = 0;
      yearWithdrawn = 0;
      yearPenaltyBase = 0;
      yearTradWithdrawn = 0;
      // Update tradBalStartOfYear AFTER the top-up so next year uses the correct Dec 31 balance
```

### Type consistency

- `getRmdFactor(age)` returns `number | null` — used with `!= null` check in engine. Consistent.
- `rmdRequired` is 0 when age < 73 (because `getRmdFactor` returns null → `rmdRequired = 0`). Table renders `—` when `rmdRequired === 0`. Consistent.
- `rmdActual` is 0 when no top-up needed. Table renders `—` when `rmdActual === 0`. Consistent.
- `yearTradWithdrawn` accumulates Traditional withdrawals from the existing withdrawal loop (same pattern as `yearOrdinaryIncome`). Reset alongside other year accumulators. Consistent.
- `tradBalStartOfYear` is a `let` declared before the month loop, updated at year boundary after top-up. Mirrors existing `rothBasis` pattern. Consistent.
